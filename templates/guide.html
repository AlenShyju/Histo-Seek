Introduction

The Browser History Analyzer is an open-source forensic tool designed to extract, analyze, and visualize browser history data with professional-grade accuracy. It helps users investigate digital footprints, uncover potential security threats, and generate detailed reports.

Purpose

This tool is ideal for:

Cybersecurity Professionals

Identify malicious browsing activity and security policy violations

Digital Forensics

Recover and analyze browser history for investigations

Privacy Audits

Understand and document browsing behavior

  • GeoIP Database: GeoLite2-City.mmdb (download from MaxMind)
  • VirusTotal API Key: Optional, for URL scanning (sign up at virustotal.com)
1

Clone the Repository

git clone https://github.com/Alenshyju/Histo-Seek.git
cd browser-history-analyzer
Note: If the repository link https://github.com/Alenshyju/Histo-Seek.git is not working, please visit https://github.com/alenshyju and search for the Histo-Seek repository directly. .
2

Install Dependencies

pip install flask geoip2 requests reportlab numpy scikit-learn
3

Set Up GeoIP Database

  1. Download GeoLite2-City.mmdb from MaxMind
  2. Place it in the project root directory
  3. Or specify its path in app.py
4

Set VirusTotal API Key (Optional)

Windows
macOS/Linux
set VIRUSTOTAL_API_KEY=your-api-key
export VIRUSTOTAL_API_KEY='your-api-key'
Note: Sign up for a free VirusTotal API key at virustotal.com
5

Run the Application

python app.py

The app will start at http://127.0.0.1:5000. Open this URL in your browser.

Finding Browser History Files

Google Chrome
Mozilla Firefox

Chrome stores browsing history in an SQLite database named History.

Windows
macOS
Linux
C:\Users\<YourUsername>\AppData\Local\Google\Chrome\User Data\Default\History
Example: C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\History
Important: Replace <YourUsername> with your Windows username. For multi-profile setups, check subfolders like Profile 1, Profile 2, etc.
~/Library/Application Support/Google/Chrome/Default/History
~/.config/google-chrome/Default/History

Firefox stores browsing history in an SQLite database named places.sqlite.

Windows
macOS
Linux
C:\Users\<YourUsername>\AppData\Roaming\Mozilla\Firefox\Profiles\<random>.default-release\places.sqlite
Example: C:\Users\alens\AppData\Roaming\Mozilla\Firefox\Profiles\abcdef12.default-release\places.sqlite
~/Library/Application Support/Firefox/Profiles/<random>.default-release/places.sqlite
~/.mozilla/firefox/<random>.default-release/places.sqlite

Steps to Locate Files

1

Close the Browser

Ensure Chrome or Firefox is completely closed to avoid file locking issues.

2

Navigate to the Path

  • Windows: Open File Explorer and paste the path
  • macOS: Use Finder's "Go to Folder" (Cmd+Shift+G)
  • Linux: Use file manager or terminal
3

Copy the File

Copy History (Chrome) or places.sqlite (Firefox) to a safe location (e.g., Desktop) to avoid modifying the original.

Cache Directories (Optional)

For analyzing potential incognito browsing data, you may need the browser's cache directory:

Chrome Cache
Windows: C:\Users\<YourUsername>\AppData\Local\Google\Chrome\User Data\Profile 3\Cache
macOS: ~/Library/Caches/Google/Chrome/Default/Cache
Linux: ~/.cache/google-chrome/Default/Cache
Firefox Cache
Windows: C:\Users\<YourUsername>\AppData\Local\Mozilla\Firefox\Profiles\abcdef12.default-release\cache2
macOS: ~/Library/Caches/Firefox/Profiles/<random>.default-release/cache2
Linux: ~/.cache/mozilla/firefox/<random>.default-release/cache2

1

Upload Browser History

  1. Find the History or places.sqlite file as described above
  2. In the "Upload History" section, click "Choose Browser History File"
  3. Select the copied history file
  4. The file name will turn green, indicating successful selection
Note: e.g C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\History
2

Analyze History

Click "Analyze History" to begin processing. A loading spinner will appear, followed by a "Analysis complete!" message when finished.

Note: Incognito data is not stored in history databases and may only be partially recovered from cache files.
3

View Results

The browsing history analysis table includes these columns:

Column Description Example

Status Active - Visible in browser history

Download History Analysis

The tool extracts and analyzes file download history with these details:

Column Description Example
File Name Name of downloaded file document.pdf
Download Time Date and time of download 2025-04-17 09:15:00
Size File size in MB 2.34 MB
Source Referrer URL where download originated https://example.com/download
Status Download completion status Completed
Browser Support:
  • Chrome downloads are fully supported (from downloads table)
  • Firefox downloads are partially supported (file name and time, but size may be unavailable)

Session Data Analysis

The tool reconstructs browsing sessions to understand user navigation patterns:

Column Description Example
URL Visited web address https://example.com/page
Referrer Source URL or "Direct" if none https://google.com
Visit Time Date and time of the visit 2025-04-17 11:22:00
Session ID Identifies related visits in the same session Session 5
Purpose: Shows navigation patterns (e.g., moving from https://google.com to https://example.com) and helps reconstruct user browsing sessions.
1

Scan URLs for Threats

  1. In the "URL Scanner" section, input a URL (e.g., https://example.com)
  2. Click "Scan"
  3. The tool queries VirusTotal API and displays results

Scan Results Include:

  • Threat Level: Low/Medium/High
  • VirusTotal Results: Malicious, Suspicious, Harmless counts
  • Last Analysis Date: When the URL was last scanned
  • Geolocation: Country, city, coordinates if available
2

Manage Suspicious Keywords

Customize the threat detection by managing keywords:

Add Keyword

  1. In the "Threat Detection" section, enter a keyword (e.g., "malware")
  2. Click "Add"
  3. The keyword is added to the list and used to flag URLs

Remove Keyword

Click the next to a keyword (e.g., "login") to remove it from detection.

Common Keywords: "login", "bank", "phishing", "scam", "password", "creditcard"

Exporting Results

1

Export as CSV

  1. In the "Browsing History Analysis" section, click "Export as CSV"
  2. The file browser_history_analysis.csv will download

CSV File Contains:

  • History Data: URL, title, visits, last visit, suspicious status, country, city, status
  • Download Data: File path, download time, size, referrer, status
2

Download Scan Reports

For individual URL scans:

  1. After scanning a URL, click "Download Scan Report"
  2. A detailed report of the scan results will be saved

Troubleshooting

File Upload Fails
  • Ensure the browser is closed before copying History or places.sqlite
  • Check the file path (e.g., C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\History)
  • Verify the file is not corrupted - try opening it with an SQLite browser
  • Check permissions - ensure you have read access to the file
No Deleted Entries Recovered
  • Deleted data may have been overwritten - try analyzing a recent database copy
  • Check terminal logs for recovery details (e.g., "Recovered 5 deleted entries")
  • Heavy browser usage increases chance of deleted data being overwritten
  • Try analyzing cache files for potential incognito browsing data
VirusTotal Scan Fails
  • Ensure the API key is set (VIRUSTOTAL_API_KEY environment variable)
  • Check the daily limit - free accounts have restrictions (4 requests/minute, 500/day)
  • Verify internet connection - the tool needs to connect to VirusTotal's API
  • Check VirusTotal status page - the service might be down
Cache Directory Invalid
  • Verify the path (e.g., C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\Cache)
  • Ensure the directory exists - cache might be in a different location
  • Check permissions - you need read access to the cache directory
  • Note: Cache analysis is optional and may not recover all incognito data

Limitations

Incognito Data

Incognito/private browsing data is not stored in history databases and can only be partially recovered from cache files.

Firefox Downloads

Size data may be unavailable due to database limitations in Firefox.

Deleted Data Recovery

Success depends on whether deleted entries remain in the database; heavy browser usage may overwrite data.

Unsupported Browsers

Currently supports Chrome and Firefox; Edge and Safari are not yet implemented.

Example Workflow

Scenario

You want to analyze Chrome history for a user named "alens" to investigate potential security policy violations.

1

Copy History File

C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\History

Copy this file to your Desktop for analysis.

2

Note Cache Path (Optional)

C:\Users\alens\AppData\Local\Google\Chrome\User Data\Profile 3\Cache

This may help recover some incognito browsing data.

3

Run Analysis

  1. Open http://127.0.0.1:5000
  2. Upload the History file
  3. Enter the cache path (optional)
  4. Click "Analyze History"
4

Review Results

  • Check for deleted entries to suspicious sites
  • Examine download history for unauthorized files
  • Scan suspicious URLs with VirusTotal integration
  • Export results as CSV for documentation

Outcome

Identify deleted visits to suspicious sites, check downloads from Profile 3, and flag URLs that violate security policies. Export comprehensive reports for documentation and further action.
